# Kodelister og korleis bruke dei

For fleire av felta i [DCAT-AP-NO](https://data.norge.no/specification/dcat-ap-no) må du peike til ein kode frå eit kontrollert vokabular. For eksempel må du velje ein kode frå EU sitt vokabular _Data Theme_ når du skal angi tema for eit datasett med eigenskapen `dcat:theme`. Kodane er organiserte i lister eller hierarkiske strukturar, og er namngitte med ein URI som ein kvar annan RDF-ressurs.

<Alert size='sm'>
    **Tips:** Standarden gir ei fullstendig oversikt over kva for kontrollerte vokabular som [**skal**
    brukast](https://data.norge.no/specification/dcat-ap-no#Kontrollerte-vokabularer-som-skal-brukes), og kva for nokre
    som [**bør** og **kan**
    brukast](https://data.norge.no/specification/dcat-ap-no#Kontrollerte-vokabularer-som-b%C3%B8r-og-kan-brukes).
</Alert>

## Korleis angi tema

Datasett og Datateneste bruker eigenskapen `dcat:theme` for å angi eitt eller fleire tema som er relevante for datasettet.

For datasett **skal** ein angi ein kode frå eit av EU sine vokabular _Data Theme_.
Ein **bør** også angi kode frå _Los_. I tillegg kan du bruke kodar frå andre vokabularer som _EuroVoc_ frå EU, men dette er valfritt.

<Alert size='sm'>
    **Merk**: I eksempla under viser vi berre bruk av kodar frå eitt vokabular om gongen, men du kan sjølvsagt bruke
    kodar frå fleire vokabular samtidig.
</Alert>

Dette er ei relativt lita kodeliste som inneheld nokre svært generelle tema.

**Døme på kodar frå Data Theme**

| Kode | Forklaring                   | URI til bruk i beskriving                                          |
| ---- | ---------------------------- | ------------------------------------------------------------------ |
| ECON | _Økonomi og finans_          | `http://publications.europa.eu/resource/authority/data-theme/ECON` |
| EDUC | _Utdanning, kultur og sport_ | `http://publications.europa.eu/resource/authority/data-theme/EDUC` |
| TECH | _Vitskap og teknologi_       | `http://publications.europa.eu/resource/authority/data-theme/TECH` |

#### Døme på bruk

```turtle
@prefix dcat: <http://www.w3.org/ns/dcat#> .

# Eit datasett som handlar om økonomi og finans
<https://example.org/datasett1> a dcat:Dataset ;
    dcat:theme <http://publications.europa.eu/resource/authority/data-theme/ECON> .

# Eit datasett som handlar om utdanning
<https://example.org/datasett2> a dcat:Dataset ;
    dcat:theme <http://publications.europa.eu/resource/authority/data-theme/EDUC> .

# Eit datasett som handlar om utdanning i tillegg til teknologi
<https://example.org/datasett3> a dcat:Dataset ;
    dcat:theme  <http://publications.europa.eu/resource/authority/data-theme/EDUC> ,
                <http://publications.europa.eu/resource/authority/data-theme/TECH> .
```

## Korleis angi dekningsområde

Vi bruker eigenskapen `dct:spatial` for å angi eit dekningsområde, altså eit geografisk område datasettet dekkjer. Dersom verdien finst i eit av EU sine vokabular, skal desse brukast, dersom ikkje, skal ein bruke verdiar frå _GeoNames_.

### EU sine vokabular

| Informasjon om vokabularet                                                                                                                                   | Oversikt over alle kodane                                                                                                                                                | URI du skal bruke i beskrivinga di                                |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------- |
| [Kontinent (op.europa.eu)](https://op.europa.eu/en/web/eu-vocabularies/dataset/-/resource?uri=http://publications.europa.eu/resource/dataset/continent)      | [Kontinent (op.europa.eu)](https://op.europa.eu/en/web/eu-vocabularies/concept/-/resource?uri=http://publications.europa.eu/resource/authority/continent)                | `http://publications.europa.eu/resource/authority/continent/XXXX` |
| [Land og territorium (op.europa.eu)](https://op.europa.eu/web/eu-vocabularies/dataset/-/resource?uri=http://publications.europa.eu/resource/dataset/country) | [Land og territorium (op.europa.eu)](https://op.europa.eu/en/web/eu-vocabularies/concept-scheme/-/resource?uri=http://publications.europa.eu/resource/authority/country) | `http://publications.europa.eu/resource/authority/country/XXX`    |
| [Stader (op.europa.eu)](https://op.europa.eu/en/web/eu-vocabularies/dataset/-/resource?uri=http://publications.europa.eu/resource/dataset/place)             | [Stader (op.europa.eu)](https://op.europa.eu/en/web/eu-vocabularies/concept-scheme/-/resource?uri=http://publications.europa.eu/resource/authority/place)                | `http://publications.europa.eu/resource/authority/place/XXXX`     |

#### Døme på bruk

```turtle
@prefix dcat: <http://www.w3.org/ns/dcat#> .
@prefix dct:  <http://purl.org/dc/terms/> .

# Eit datasett som har Europa som dekningsområde
<https://example.org/datasett1> a dcat:Dataset ;
    dct:spatial <http://publications.europa.eu/resource/authority/continent/EUROPE> .

# Eit datasett som har Noreg som dekningsområde
<https://example.org/datasett2> a dcat:Dataset ;
    dct:spatial <http://publications.europa.eu/resource/authority/country/NOR> .

# Eit datasett som har staden Tromsø som dekningsområde
<https://example.org/datasett3> a dcat:Dataset ;
    dct:spatial <http://publications.europa.eu/resource/authority/place/NOR_TOS> .
```

## Korleis angi lisens

Vi bruker eigenskapen `dct:license` for å angi kva lisens som gjeld for ein katalog, eit datasett, ei datateneste eller ein distribusjon. Verdien skal veljast frå EU sitt kontrollerte vokabular _License_.

| ------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------- |
| [Gå til op.europa.eu](https://op.europa.eu/en/web/eu-vocabularies/dataset/-/resource?uri=http://publications.europa.eu/resource/dataset/licence) | [Gå til op.europa.eu](https://op.europa.eu/en/web/eu-vocabularies/concept-scheme/-/resource?uri=http://publications.europa.eu/resource/authority/licence) | `http://publications.europa.eu/resource/authority/licence/XXXX` |

**Døme på kodar frå License**

| Kode        | Forklaring                                                | URI til bruk i beskriving                                              |
| ----------- | --------------------------------------------------------- | ---------------------------------------------------------------------- |
| CC0         | _Creative Commons CC0 1.0 Universal_                      | `http://publications.europa.eu/resource/authority/licence/CC0`         |
| CC_BY_4_0   | _Creative Commons Attribution 4.0 International_          | `http://publications.europa.eu/resource/authority/licence/CC_BY_4_0`   |

## Korleis angi format

Vi bruker eigenskapen `dct:format` for å angi kva format ei datateneste eller ein distribusjon tilbyr. Verdien skal veljast frå EU sine kontrollerte vokabular _File Type_.

| Informasjon om File type                                                                                                                           | Oversikt over alle kodane                                                                                                                                   | URI du skal bruke i beskrivinga di                                |
| -------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| [Gå til op.europa.eu](https://op.europa.eu/en/web/eu-vocabularies/dataset/-/resource?uri=http://publications.europa.eu/resource/dataset/file-type) | [Gå til op.europa.eu](https://op.europa.eu/en/web/eu-vocabularies/concept-scheme/-/resource?uri=http://publications.europa.eu/resource/authority/file-type) | `http://publications.europa.eu/resource/authority/file-type/XXXX` |

**Døme på kodar frå File type**

| Kode | URI til bruk i beskriving                                         |
| ---- | ----------------------------------------------------------------- |
| JSON | `http://publications.europa.eu/resource/authority/file-type/JSON` |
| XML  | `http://publications.europa.eu/resource/authority/file-type/XML`  |
| CSV  | `http://publications.europa.eu/resource/authority/file-type/CSV`  |
| YAML | `http://publications.europa.eu/resource/authority/file-type/YAML` |
| TXT  | `http://publications.europa.eu/resource/authority/file-type/TXT`  |
| PDF  | `http://publications.europa.eu/resource/authority/file-type/PDF`  |

#### Døme på bruk

Med distribusjon:

```turtle
@prefix dcat: <http://www.w3.org/ns/dcat#> .
@prefix dct:  <http://purl.org/dc/terms/> .

# Eit datasett som distribueres i et CSV-format
<https://example.org/datasett1> a dcat:Dataset ;
    dcat:distribution <https://example.org/distribution1> .

<https://example.org/distribution1> a dcat:Distribution ;
    dct:format <http://publications.europa.eu/resource/authority/file-type/CSV> .
```

Med datateneste:

```turtle
@prefix dcat: <http://www.w3.org/ns/dcat#> .
@prefix dct:  <http://purl.org/dc/terms/> .

# Datasett tilgjengeliggjort med et API som tilbyr JSON
<https://example.org/datasett1> a dcat:Dataset .

<https://example.org/dataservice1> a dcat:DataService ;
    dcat:servesDataset <https://example.org/datasett1> ;
    dct:format <http://publications.europa.eu/resource/authority/file-type/JSON> .
```
