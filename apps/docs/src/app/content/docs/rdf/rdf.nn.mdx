# Kva er Resource Description Framework og lenka data?

Om du har komme hit, er det kanskje fordi du lurer på kva alt snakket om RDF (_Resource Description Framework_) eigentleg dreier seg om.

## Kva er RDF?

Så kva er då eigentleg RDF? Fritt oversatt betyr _Resource Description Framework_ noko slikt som _rammeverk for beskrivelse av ressursar_, og dette forklarer eigentleg heile RDF:

- ein _ressurs_ kan vere kva som helst, inkludert dokument, nettsider, folk, fysiske objekt, og abstrakte konsept og idear.
- med RDF _beskriver_ ein ressursane og relasjonane mellom dei.
- RDF er eit _rammeverk_ fordi
  - det er eit formelt og logisk språk,
  - det er ein modell for å strukturere data, og
  - det inkluderer verktøy og algoritmar for å behandle dataen.

Så med RDF kan ein beskrive _ting_ og korleis desse _tinga_ forholdar seg til kvarandre.
Sidan RDF er basert på web-teknologi kan du knyte dine beskrivingar til andres, og vi snakkar då om lenka data.

## Kvifor RDF?

Det er nokre grunnar til at vi bruker akkurat RDF i beskrivingane som blir publisert til data.norge.no.
RDF har nemleg nokre fordelar som gjer at det egnar seg godt til ein datakatalog. Nokre av desse er:

1. Samhandlingsdyktighet (eller interoperabilitet):

   - a. RDF har eit globalt namnrom, som betyr at vi kan snakke om dei same tinga på tvers av system.
   - b. Ulike beskrivingar (som datasett og omgrep) kan enklare knytast saman (lenka data).
   - c. RDF gjer det enkelt å gjenbruke vokabularar og informasjon på tvers av system og løysingar.

2. RDF er maskinlesbart og kan utvekslast og forståast av ulike datasystem.
3. Semantikken følger med dataen når ein bruker klassar og eigenskapar definert i RDF; dataen blir sjølvforklarande.
4. RDF kan handtere kompleks data som er vanskeleg å definere i tabellform.
5. RDF gjer distribuerte spørringar mogleg.
6. Ein stor verktøykasse er tilgjengeleg (for modellering, validering, spørringar, distribuert data, med meir).

Data.norge.no hentar beskrivingar frå mange ulike verksemder, og då er det ein fordel at vi kan bruke eit maskinlesbart og standardisert format som RDF.

Den europeiske dataportalen [data.europa.eu](data.europa.eu) er basert på RDF og hentar beskrivingar frå blant anna data.norge.no. Det betyr at data.norge.no må tilby den i RDF.

Alle beskrivingane av datakatalogar til både data.norge.no og [data.europa.eu](https://data.europa.eu) følger ein standard som heiter [DCAT (Data Catalog Vocabulary)](https://www.w3.org/TR/vocab-dcat-3/) som er definert i RDF.
