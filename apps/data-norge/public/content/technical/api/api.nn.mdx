# API

<Ingress size='xs'>Her finn du informasjon om endepunkta du kan nytte til å hente ut data.</Ingress>

Det er to forskjellige måtar å søke etter data på Data.norge.no gjennom endepunkt. Dei er basert på kvar sin teknologi. Utover desse tilbyr vi eit oppslags-endepunkt kor ein kan hente ut eiga verksemd sine beskrivingar frå registreringsløysinga. 

## Søk etter data på Data.norge.no

### Søk

Søkemotoren på Data.norge.no byggjer på Elastic search. Denne tenesten søker i tekst i eit avgrensa antal felt og har filtre og anna avansert funksjonalitet som hjelper brukarane med å finne fram.

[Les meir om søkeendepunktet (på engelsk)](/docs/technical-documentation/endpoints/search).

### SPARQL

SPARQL-søket byggjer på rdf-spørrespråket SPARQL, og er det mest avanserte og kraftfulle søket vårt. Brukaren er nøydd til å lage spørringar som følgjer rett syntaks, men vil kunne søke i alle datapunkt som er hausta, ikkje berre førehandsvalgde felt. Endepunket er opent for oppslag, men ein kan ikkje skrive til det.

[Les meir om SPARQL-endepunktet (på engelsk)](/docs/technical-documentation/endpoints/sparql).

## Hent ut data frå registreringsløysinga. 

### Catalog View

Verksemder som nyttar registreringsløysinga kan hente ut deira eigne beskrivingar. Førebels er det berre mogleg å hente ut omgrep frå registreringsløysinga. API-et er beskytta med Maskinporten. 

[Les meir om Oppslags-endepunktet mot registreringsløysinga (på engelsk)](/docs/technical-documentation/endpoints/catalog-view).
